Welcome to Beryl's Todo List API

This page contains some simple ways to get information from a todos list by accessing different http pathways on the localhost 8080.

  • GET/

    Returns the index.html (this page).

  • GET/todos

    The :id referes to the id in the todos.json file, in the api replace the :id including the semi-colon

    Returns a list of todos with the id, name, due date, when it was created and if it is completed from the todos.json

    Response Body example:

    {
    "id": "19d539a11189-4a60-3a4c-4434-01507581",
    "name": "Learn to juggle",
    "created": "2021-11-23T14:35:01.752Z",
    "due": "2021-10-20T18:25:43.511Z",
    "completed": false
    } ,

  • GET/todos/:id

    Returns a specific todo with the corresponding id by the client.
    The API should return the matching todo with the id, name, due date, when it was created and if it is completed, e.g.

    Request URL:

    http://localhost:8080/todos/19d539a11189-bb60-u663-8sd4-01507581

    Response Body:

    {
    "id": "19d539a11189-bb60-u663-8sd4-01507581",
    "name": "Buy 2 Cartons of Milk",
    "created": "2021-10-20T18:25:43.511Z",
    "due": "2021-11-23T23:05:03.352Z",
    "completed": true
    }

  • GET/todos/overdue

    Returns the todos that are overdue based on the due attribute with the id, name, due date, when it was created and if it is completed,
    e.g.

    Request URL:

    http://localhost:8080/todos/overdue

    Response Body:

    {
    "id": "19d539a11189-4a60-3a4c-4434-01507581",
    "name": "Learn to juggle",
    "created": "2021-11-23T14:35:01.752Z",
    "due": "2021-10-20T18:25:43.511Z",
    "completed": false
    }

  • GET/todos/completed

    Returns a list of completed todos, for example

    Request URL:

    http://localhost:8080/todos/completed

    Response Body:


    {
    "id": "19d539a11189-bb60-u663-8sd4-01507581",
    "name": "Buy 2 Cartons of Milk",
    "created": "2021-10-20T18:25:43.511Z",
    "due": "2021-11-23T23:05:03.352Z",
    "completed": true
    }

  • POST/todos

    Post a new todo object to server in the same format as the previous examples using postman.
    If successful, you will see “Created new Todo and the status 200. If unsuccessful it will send “Incorrect data submitted” and status 400.
    For example:

    Structure to add a todo:

    {
    "id": “create an id”
    "name": ". . . . . . . ",
    "created": ". . . . . . ",
    "due": ". . . . . . . . ",
    "completed": “can be ONLY BE true or false
    }

  • POST/todos/:id/complete

    Updates the specific todo by updating the completed to true. If updated the status code will be 200, if not 400 will be returned.

  • POST/todos/:id/undo

    Updates the specific todo by updating the completed to false. If updated the status code will be 200, if not 400 will be returned.

  • PATCH/todos/:id

    Edits the name, due date, or both attributes specific to the todo being requested. The id of the todo will remain the same but the name and/or due attribute will change. It will return status 200 and success if successful or 400 if the id doesn't exist.
    You must follow the same structure as the example in the post request but only using the “name” and/or “due” attributes. e.g:

    Request URL:

    http://localhost:8080/todos/19d539a11189-4a60-3a4c-4434-01507581

    Request Body:

    {
    "name" : ". . . . ",
    "due": “. . . . . . ”
    }

  • DELETE/todos/:id

    Deletes the todo by the specificid.
    It will returns 200 if deleted successfully and 400 if unsuccessfully deleted.
